Blockaid — Transaction Validation Flow

dApp → Wallet → Blockaid → user sees verdict → signs (or doesn't)
dApp (origin)
Wallet (UI + signer)
Blockaid (validation engine)
Node (execution / state)
Blockchain
EOA (target address under review)

The wallet connects to the dApp and receives a transaction object (from, to, value). Before prompting the user to sign, the wallet sends the transaction and origin dApp to Blockaid. Blockaid simulates the transaction against on-chain state (via a node / fork), enriches it with address reputation and dApp scanning signals, and returns a verdict and asset-diff. The wallet renders the verdict — here, Malicious (+1 ETH, -2000 USDC) — so the user can decide before the signed transaction ever hits the blockchain.
